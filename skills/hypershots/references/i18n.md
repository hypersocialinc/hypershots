# i18n contracts

The exact marker, schema, fit, and layout contracts behind the translate gear. `translate.md` is the runbook; this is the spec.

`<skill>` = installed skill dir, `<ws>` = your workspace (default `.shots`).

## data-i18n markers

- One marker per translatable text node: `<div class="headline" data-i18n="p1.headline">…</div>`.
- **Double quotes only.** The injector regex matches `data-i18n="…"`; single quotes are silently invisible — English ships.
- Element content: text + `<br>` only (self-closing `<br/>` tolerated). Any other nested tag inside a marked element is a fatal inject error. Style fragments as sibling marked elements instead of inline `<b>`/`<span>`.
- Keep the whole element on one line in the HTML — easier to grep, no whitespace surprises in the replacement.

## strings.<locale>.json schema

```json
{ "locale": "es", "strings": {
  "p1.eyebrow": "Antes de ir",
  "p1.headline": "¿Qué tan limpio es<br>tu restaurante favorito?",
  "p1.sub": "Quizá no quieras saberlo."
} }
```

- Top-level `strings` object is required; the injector exits 1 without it.
- Key naming: `p<N>.<slot>` — `p1.headline`, `p3.sub`, `p2.chip`. Panel-scoped so a grep of one panel shows exactly its keys.
- Exact bijection with markers: missing key = fatal, unused key = fatal (see translate.md).

## Auto-fit mechanism (fit.js)

Runs in-page during render; included LAST in `<body>`.

1. Awaits `document.fonts.ready` — fit never measures fallback-font metrics.
2. Reads `--panel-w`, `--panel-h`, `--device-top-ratio` from `:root`. If any is not finite (frame.css/profile.css not linked) it records a `frame-vars-missing` fit failure — render exits 2 rather than producing an unmeasured panel.
3. For each `[data-fit]` element: budget = `data-fit-max` px if set, else device-top − 14. Decrements the element's font-size 1px at a time until the copy block — the element **and all following siblings** (`.sub`, etc.) — has bottom ≤ budget. Shrinking the headline reflows the block, pulling trailing siblings up out of the device zone; that's why the failure mode being guarded is *overlap with the device*, not page overflow.
4. Floor: `data-fit-floor` px, default **26**. If the block still doesn't clear the budget at the floor, a fit failure is recorded (named by `data-i18n`, falling back to class name) and render exits **2** — the copy needs a rewrite, not a smaller floor.
5. Dumps `{panelW, panelH, fitFailures, boxes}` (boxes = `getBoundingClientRect` of every `[data-protect]` element) as `<script type="application/json" id="hypershots-boxes">`; render.sh extracts it into `panel-N.boxes.json`.

Fit is deterministic across render's two Chrome passes — which is also why panels must not depend on time, randomness, or animation.

## Output layout

```
<ws>/panels/                    # source-language panels (rendered as locale "en")
<ws>/panels-<locale>/           # GENERATED by translate-inject — never hand-edit
<ws>/strings.<locale>.json      # authored, per locale
<ws>/out/<profile>/<locale>/    # panel-N.png + panel-N.boxes.json (+ chrome.log on failure)
```

- `render.sh <ws> <profile> en` reads `panels/`; any other locale reads `panels-<locale>/`.
- `panels-<locale>/` is regenerated (and stale files purged) on every successful inject. Edit the source panel or the strings file, never the generated copy.
- Render purges `out/<profile>/<locale>/panel-*` before rendering — outputs always reflect the current panel set.

## Font coverage (vendored, fonts.css)

| Family | Weights/styles | Subsets |
|---|---|---|
| Inter Tight | variable 400–900, normal + italic | latin, latin-ext |
| IBM Plex Mono | 400, 500, 600 | latin, latin-ext |

- **Covered:** Western European (en/es/fr/de/it/pt/nl/…), Central/Eastern European Latin — Turkish, Polish, Czech, Slovak, Romanian, Hungarian, Croatian; Vietnamese partially (latin-ext lacks some stacked-diacritic precomposed glyphs — Read the rendered PNG before shipping vi).
- **Not covered:** Cyrillic, Greek, Arabic/Hebrew (also need RTL `dir` handling), CJK, Thai, Indic. Uncovered glyphs fall back per-glyph to system fonts — mixed faces mid-word. Roadmap: Noto fallback stack. Don't ship these locales on the vendored fonts.
- `font-display:block` + local files: no network fetch, no FOUT race against Chrome's `--virtual-time-budget`.
